# Stryde Brand Identity & Visual Guidelines
> **Local Claude Code Orchestration**

Stryde is an approachable, structural, and high-performance desktop control panel designed to abstract the complexity of Claude Code configurations (agents, skills, MCP servers, hooks, and plugins) out of raw markdown files and into a guided visual environment.

---

## 1. Core Brand "Vibe"
Stryde sits at a unique intersection of heavy-duty engineering utility and human-centric software.
* **Accessible Power:** It honors the underlying technical capability without intimidating the user.
* **Tactile & Architectural:** Elements feel like structural blocks, layers, and guided paths, emphasizing project levels.
* **Approachable Utility:** Inspired by Claude’s elegant, organic interface, Stryde utilizes softened serifs or humanistic sans-serif type treatments alongside rounded containers to ground the technical power into a casual, comfortable workspace.

---

## 2. Naming & Domain Architecture
* **Name:** Stryde
* **Metaphor:** Finding your flow; skipping over the boilerplate file-IO setup and immediately moving forward ("hitting your stride") with multi-agent orchestration.
* **Domain:** `stryde.dev`

---

## 3. Brand Assets & Visual Elements

### Logo Design: The Isometric Path
The official logo features an interlocking, three-dimensional isometric staircase pattern using alternating colored blocks. 
* **Symbolism:** Represents project levels (global configurations down to directory-level overrides), structured stacks, and moving step-by-step through advanced automated workflows.
* **Execution:** Clean geometric shapes with subtle highlights, easily identifiable as a desktop app icon or a navigation header mark.

### Typography
To balance high-performance local execution with a casual, human feel (inspired by modern conversational AI platforms):
* **Logotype / Headers:** A humanistic, slightly rounded sans-serif or crisp geometric face (e.g., *Geist*, *Inter*, or *SF Pro Rounded*).
* **Body / Interface Text:** A highly legible, approachable corporate serif or highly readable text face that feels friendly and warm, avoiding rigid monochromatic monospace layouts for standard configurations.

## 4. The Cockpit Color Palette

| System Layer | Color Name | Hex Code | Purpose / Visual Role |
| :--- | :--- | :--- | :--- |
| **Canvas** | Deep Charcoal | `#0B0F19` | Soft, immersive dark background reducing strain during long local sessions. |
| **Panels & Cards** | Obsidian Gray | `#1E2230` | Structural card overlays, sidebar bases, and input borders. |
| **Execution Accent** | Soft Kinetic Teal | `#00E5FF` | Active triggers, primary actions (`+ New Agent`), and progress steps. |
| **Data Accent** | Rich Electric Violet | `#7C4DFF` | Highlight tags, selected menu categories, MCP server nodes, and visual connectors. |
| **Typography** | Off-White | `#F5F7FA` | High-contrast, clean reading surfaces across markdown code views. |

---

## 5. Value Pillars & Key Messaging
1. **Zero-Friction Contexts:** Structural management of project-level files without manually writing lines of JSON or tracking down `.claude/` states.
2. **Visual Teams & Orchestration:** Build, tweak, and link complex pipelines using guided steps that are as effortless for non-technical creators as they are fast for engineers.
3. **Local Safety & Execution:** Deeply integrated into the local system environment, working directly where your code lives with optimal fluid speed.
